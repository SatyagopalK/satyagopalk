Machine Learning

Satya Gopal · 10 min read

When people talk about Artificial Intelligence, they define it in many ways — acting like humans, thinking like humans, or behaving rationally. When I first encountered these definitions, they felt important but distant. Yet one idea kept repeating itself: intelligence is about making the right decisions.

As I learned more, I realized that Machine Learning is one of the most practical ways to achieve this intelligence. Instead of writing every rule by hand, we show machines data and allow them to learn from it.

Like most students, I memorized definitions by Arthur Samuel and Tom Mitchell during my undergraduate days just to clear exams. It was only after working on small projects that something clicked — machine learning wasn’t really about definitions or formulas. It was about patterns.

Machine learning looks at data the way a child looks at pictures first — trying to understand the story before reading the words.

To understand whether these patterns are meaningful, we evaluate them using metrics. With this perspective, those famous definitions finally started to make sense — not as exam answers, but as intuitive explanations.

Types of Machine Learning

Before discussing the types of machine learning, I’d like to acknowledge Aurélien Géron, whose work helped me understand how learning approaches are used in real-world systems.

Most books and tutorials mention four major types — Supervised Learning, Unsupervised Learning, Semi-Supervised Learning, and Reinforcement Learning. While these categories are important, it’s even more useful to understand how they differ in practice.

Based on the Amount of Supervision Given

One simple way to classify machine learning systems is by looking at how much supervision is provided during training.

Supervised Learning

Think of solving a math problem in school. You are given the question and the correct answer. Your task is to figure out the steps connecting the two.

Supervised learning works the same way. The algorithm is trained using labeled data — inputs paired with correct outputs. Once trained, the model can predict outcomes for unseen data.

Real-world examples:

  • Email spam detection
  • House price prediction

Common algorithms:

  • Linear Regression
  • Logistic Regression

Unsupervised Learning

Imagine being given a notebook full of questions but no answers. Your task is to observe patterns, group similar questions, and find structure on your own.

In unsupervised learning, the algorithm receives unlabeled data and tries to discover hidden patterns or relationships.

Real-world examples:

  • Customer segmentation
  • Clustering news articles by topic

Common algorithms:

  • K-Means Clustering
  • Hierarchical Clustering

Semi-Supervised Learning

Semi-supervised learning lies between supervised and unsupervised learning. A small portion of data is labeled, while the rest is unlabeled.

This approach is especially useful when labeling data is expensive or time-consuming.

Real-world examples:

  • Image classification with limited labeled data
  • Speech recognition systems

Reinforcement Learning

Reinforcement learning is learning through experience — like learning to ride a bicycle. You try, fail, adjust, and improve over time.

An agent interacts with an environment and receives rewards or penalties. The goal is to maximize long-term rewards.

Real-world examples:

  • Game-playing systems like AlphaGo
  • Robots learning to walk

Batch and Online Learning

Machine learning systems can also be categorized based on how they learn from incoming data.

Instance-Based vs Model-Based Learning

Instance-based learning relies on memorization and similarity, while model-based learning focuses on understanding underlying patterns and generalization.

Main Challenges of Machine Learning

  • Choosing the right algorithm
  • Insufficient data
  • Non-representative datasets
  • Poor data quality
  • Irrelevant features
